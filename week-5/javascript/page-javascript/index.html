Script Loading Strategies

All the html in the page is loaded in order which they appear.If you are using javascript to manipulate in the page (or more accurately the DON)your code wont work if the javascript is loaded and parsed before the html you are trying to do something to


They are a few different strategies to make sure your javascript only run after the html is parsed

  • In the external javascript example above which we delected,the script element is placed at the bottom of the body of the document and therefore only runs after the rest of the html is parsed
  • In the external javascript above,the script element is placed in the head of the document before the html is parsed.But because we are using script type"module"the code is treated as a module and the browser wait for all html to be processed before executed javascript module.(you can also placed external script at the bottom of the body.But if there is a lot of html and the network is slow,it may take a lot of time before the browser can even startfetching and loading the script,so placing the javascript in the head is better
  • If you still want to use the non-module script in the document head,which could block the whole page from displaying in the screen and could cause error because it execute before the html is parsed
    • For external script,you should add the defer(or if you dont need the html to be ready they are synced attribute on the script element)
    • For internal script,you should wrap the code in a domcontentloadedEventlistener
